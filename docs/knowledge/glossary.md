# Глоссарий и доменная модель

> Доменные термины наблюдаемости/трейсинга — чтобы агенты говорили на языке проекта.

- **Спан (span)** — единица трейса с началом и концом: пара событий start/end с общим `spanId`. Под-агент
  даёт спан `subagent.start`/`subagent.end`; вызов инструмента — `tool.start`/`tool.end`. Спан без `end` =
  «идёт» (`running`).
- **Лейн / дорожка (lane)** — линия группировки действий в ленте. Значения: `"orchestrator"` (действия
  оркестратора) или `spanId` под-агента. Определяется best-effort (см. ADR-0003), т.к. под-агенты делят
  один `session_id`.
- **`tool.start` / `tool.end`** — события телеметрии о начале/конце вызова значимого инструмента
  (Bash/Read/Grep/Glob/Edit/Write). Источник живой ленты `/trace/feed`. Под-агенты сюда НЕ попадают.
- **`toolUseId`** — идентификатор вызова инструмента из payload хука (`toolu_...`). Один и тот же у `Pre` и
  `Post`, совпадает с `content[].id` блока `tool_use` в транскрипте — сквозной ключ «событие ↔ строка
  транскрипта».
- **`spanId`** — стабильный ключ склейки start↔end: `"tool-" + toolUseId` для инструментов,
  `"span-" + toolUseId` для под-агентов.
- **`attributionAgent`** — роль под-агента (напр. `ai-pathfinder:wf-explorer`), проставленная в его
  assistant-записях транскрипта. Точный, но **посмертный** признак исполнителя (в payload хука его нет).
- **Курсор-оффсет (byte offset)** — позиция в `telemetry.jsonl` в **байтах** (`f.tell()`), которой
  `/trace/feed` помечает прочитанный хвост. Клиент возвращает её как `since`, чтобы дочитать только новое.
  Отличать от **курсора Langfuse** (`telemetry.cursor`) — там счётчик отправленных **строк**.
- **Delta-feed (delta-only лента)** — режим `/trace/feed`: на каждый тик отдаются только события,
  дозаписанные после `since`; сервер stateless, склейка спанов — на клиенте.
- **`running`** — состояние спана, у которого есть `start`, но ещё нет `end` (инструмент идёт или был
  прерван). Выводится клиентом, длительность не считается.
- **`telemetry.jsonl`** — append-only журнал событий задачи: `.workflow/tasks/<slug>/telemetry.jsonl`.
- **Транскрипт** — JSONL-лог сессии/под-агента от Claude Code в `~/.claude/projects/<proj>/...`; источник
  текста сообщений (`type==assistant`, блоки `text`) и чисел usage.
- **untracked** — новый, никогда не стейдженный git-файл (`status added`); во вкладке «Изменения»
  помечается полем `untracked: true`. Каталоги разворачиваются в файлы флагом `-uall`.
- **untracked-мусор** — пустой (0-байтный) untracked-файл (`-`, `Заполняется`, …), скрываемый предикатом
  `_is_noise`; реальный stray-файл, не git-артефакт (см. ADR-0005).
- **Тумблер «только tracked / все»** — segmented-переключатель в шапке вкладки «Изменения»; по умолчанию
  «только tracked» (untracked-мусор скрыт), «все» показывает весь untracked. Фильтрует на фронте по полю
  `untracked`, без повторного запроса.
- **C-quoting (git quotePath)** — экранирование git'ом не-ASCII путей в `\NNN`-вид (кириллица →
  `\320\277…`); отключается `-c core.quotePath=false` для честных UTF-8-имён.
