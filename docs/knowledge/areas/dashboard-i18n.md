# Область: Интернационализация дашборда и хаба (i18n, eng-first)

> UI дашборда и хаба — **english-first** с переключателем `en`/`ru`, управляемым **глобальной** настройкой
> плагина в `~/.claude/ai-pathfinder/settings.json` через эндпоинты `GET /settings.json` / `POST /settings`.
> Контрол — флип-иконка `#lang-btn` рядом с `#theme-btn`. Механика — клон темы (ADR-0015). Решение и
> рационал — ADR-0018. Смежная область — `areas/dashboard-theming.md` (тот же паттерн словарь/bootstrap).

## Назначение

Дать пользователю переключать язык UI дашборда и хаба между **двумя языками — English / Русский**
(флип-иконка EN⇄RU) с сохранением **глобально на весь плагин** (один язык, не per-project) и
кросс-страничным синком хаб↔задача. Дефолт — `en`. **Важно:** эта настройка управляет только **хромом UI**
и служит **фолбэком** языка вывода агента — язык вывода агента определяется **языком запроса человека**
(см. ниже «Язык агента»). Контент агента из `dashboard.json` (title/summary/questions/now) НЕ локализуется
фронтом.

## Где живёт настройка (глобальный конфиг)

- **`~/.claude/ai-pathfinder/settings.json`** — `{"lang": "en"|"ru", "ts": <iso>}`. Дефолт `en`, белый
  список `{en, ru}`, любая ошибка/неизвестное значение → деградация к `en`. **Не** в `.workflow`
  (per-project store, ADR-0010) и **не** в плагин-руте (кэш-копия, перетирается refresh; ADR-0017,
  ADR-0018).
- Хелперы `scripts/server.py`: `settings_path()` (с override-параметром базы для тестовой изоляции —
  НЕ писать в реальный `~/.claude/`), `read_lang()` (нормализация белым списком → `en`),
  `write_lang(lang)` (atomic через `Workspace.write_json`, best-effort try/except). Чтение/запись —
  через `Workspace.read_json`/`write_json` (atomic tmp+`os.replace`, UTF-8, `ensure_ascii=False`).

## Контракт эндпоинтов (единый источник истины)

- **`GET /settings.json`** — slug-free ветка в `do_GET` (рядом с `/queue.json`), **без кэша**
  (`_send` уже no-store), тело `{"lang": <en|ru>}`. **Единственный канал** для дашборда И хаба — НЕ
  дублируется в `/hub.json` (там TTL-кэш `_hub` → рассинхрон; см. ADR-0018).
- **`POST /settings`** — ветка в `do_POST` **ДО проверки slug** (критично: `do_POST` отвергает запрос
  без slug 400, а у `/settings` slug нет → ветку вставить сразу после чтения body). Валидирует `lang`
  белым списком (иначе 400), пишет atomic, возвращает `{"ok": true, "lang": lang}`. **Хаб — писатель**
  настройки.

## Словарь и `t()` (фронт)

- Инлайн-словарь `STR = { en:{...}, ru:{...} }` с **плоскими английскими ключами** (`t("tab.changes")`,
  `t("hub.activeRuns")`) + функция `t(key)` — берёт строку из активного языка. Канонический перевод
  общих лейблов — `.workflow/tasks/dashboard-i18n/i18n-glossary.md` (вкладки/кнопки/статусы; чтобы
  дашборд, хаб и README цитировали одинаковый текст).
- **Дубль словаря в двух местах:** `templates/dashboard.html` и `HUB_PAGE`/`INDEX_LANDING` в
  `scripts/server.py` (raw-строка, функции с дашбордом не шарятся — отдельные стримы, как у темы).
- Динамические строки берутся из словаря **на каждом тике поллинга** внутри рендер-функций → смена
  языка триггерит перерисовку (re-`render(lastData, …)`).

## Источник истины и синк

- **Источник истины — глобальный `/settings.json`**, `localStorage['lang']` — кэш. При старте
  `fetch('/settings.json')`; смена в хабе → `POST /settings` + `localStorage`; поллинг `/settings.json`
  (или общий ключ + тик хаба) даёт кросс-страничный синк хаб↔дашборд. Дефолт — **жёсткий `en`** (не
  `navigator.language`).
- Логика по образцу темы (ADR-0015): `storedLang()` / `resolveLang()` / `applyLang()` / `initLang()`;
  `<html lang>` + head-bootstrap резолвят язык до первой отрисовки. Состояние языка — на
  `documentElement`/модульной переменной, **вне** `#content`/`#trace-summary` (sig-гард, как тема).

## Контрол

- **Флип-иконка-кнопка `#lang-btn` (EN⇄RU)** в шапке рядом с `#theme-btn`, стиль по `.theme-btn`/`.ghost`.
  Клик флипает язык. **В хабе контрол обязателен** (бриф: «меняется из hub»); клик → `POST /settings` +
  `localStorage`. Выбрана флип-иконка (демо v1), не segmented `.seg` (демо v2).

## Язык агента (часть 3, конвенция)

Правило языка вывода агента (эволюция: жёсткое «human-facing → Russian» → eng-first + чат-исключение,
ADR-0018 → **язык запроса побеждает везде**, ADR-0022; см. `conventions.md`, `areas/orchestrator-skills.md`):

- **Язык запроса человека ПОБЕЖДАЕТ** — авто-детект из его сообщения. INTAKE-шаг скилла резолвит язык
  запроса → пишет в `state.json.lang` → передаёт под-агентам в промпте спавна (оркестратор медиирует
  хэндофф). Это язык **всего, что читает человек**: нарратив оркестратора в терминале, дашборд,
  brief/exploration/plan/questions/summary/PRD, тексты гейтов, опции choice, `replies.json`,
  `chat.jsonl` (вкл. anchored), `/ask` summary/planBlocks.
- **Глобальная настройка — только фолбэк** (`~/.claude/ai-pathfinder/settings.json`, `en` при
  отсутствии), когда человеческого запроса нет (autonomous/eval-прогоны). И отдельно — это язык хрома UI.
- **Всегда английский** (если человек явно не попросил иначе): `docs/knowledge/**`, README, git-коммиты.
- **Стабильно английские:** машинно-парсимые заголовки/ключи (секции `ask-researcher`, `cand:`-ключи
  `wf-improver`, схемы дайджестов) — не переводятся никогда, иначе ломается парсер оркестратора.

## Инварианты

- **Оба языка содержат одинаковый набор ключей.** Ключ, добавленный в `STR.en`, **обязан** быть в
  `STR.ru` (иначе `undefined`). То же для дубля в `dashboard.html` ↔ `HUB_PAGE`. Закреплено
  **smoke-тестом полноты словаря** (по аналогии с инвариантом полноты тёмной палитры, ADR-0015).
- **`/settings.json` — единственный канал языка**, без кэша; в `/hub.json` язык НЕ дублируется.
- **`POST /settings` обходит проверку slug** (ветка до неё) — иначе 400 на валидном запросе.
- **Резолв/дефолт продублирован** (head-bootstrap ↔ `<body>`-функция; дашборд ↔ хаб) — правки синхронны.
- **Контент агента не локализуется фронтом** — `t()` только для хрома UI.

## Подводные камни

- **Риск дрейфа словаря** — главная ловушка: две копии `STR` (дашборд/хаб) легко рассинхронить, как
  именную шероховатость `--warn-soft`/`--awaiting-soft` темы. Smoke-тест ловит рассинхрон ключей.
- **`index.html` — per-task КОПИЯ шаблона** (`server.py:178/193`), не сам `templates/dashboard.html`.
  Правки шаблона видны только в НОВЫХ задачах; для текущей нужна повторная копия шаблона в `index.html`.
- **Плагин-кэш.** Живой companion-сервер может крутить python из plugin-CACHE; серверные правки
  (`/settings*`, словарь хаба) видны после bump/refresh. Для разработки запускать `scripts/server.py`
  из РЕПО напрямую. Поведение агента (часть 3) применяется к рантайму **только после refresh**.
- **Самый плотный блок локализации — «Трейсинг»** (легенды/метрики/тултипы/фазы/единицы `fmtDur`):
  легко пропустить строку.
- **`want_role in (..., "оркестратор")` (`server.py:557`) НЕ трогать** — это значение роли из
  телеметрии, а не UI-лейбл (UI-лейбл `orchestrator` переводится отдельно в словаре).
- **Гонка записи `settings.json`** — несколько серверов пишут один файл, last-writer-wins, `os.replace`
  атомарен, лок не нужен (гонка = редкий ручной клик).

## Как расширять

- **Новая UI-строка:** добавить ключ в **обе** вилки `STR.en`/`STR.ru` **в обоих файлах** (дашборд + хаб,
  если применимо), использовать через `t("...")`. Smoke-тест требует полноты.
- **Новый язык (>2):** пополнить `STR` третьим кодом + белый список `{en, ru, ...}` + smoke-тест; но
  флип-иконка рассчитана на 2 языка — для N нужен segmented/dropdown контрол.
- **Сменить дефолт первого захода:** правится ветка резолва «настройки нет» в head-bootstrap **и** в
  `resolveLang()` (обе копии — дашборд и хаб); сейчас жёсткий `en`.
- **Новое поле глобальной настройки:** добавить в `settings.json` (тот же файл), отдавать через
  `/settings.json`, писать через расширение `POST /settings`; не дублировать в кэшируемый `/hub.json`.
