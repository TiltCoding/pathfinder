# Область: Темизация дашборда (светлая/тёмная тема)

> Пользовательское переключение темы дашборда и хаба: предпочтение в `localStorage`, разрешённая тема
> на `data-theme`-атрибуте `<html>`, FOUC-bootstrap до первой отрисовки. Решение и рационал — ADR-0015.

## Назначение

Дать пользователю выбирать тему (Светлая / Тёмная / Системная) с сохранением между сессиями и без
вспышки неправильной темы при загрузке. Тема — **чисто клиентская**: сервер и контракты данных
(`dashboard.json`/`/data`/`/trace`/`/changes`/`/hub.json`) не меняются.

## Где живут токены (3 `:root`-блока + хаб)

Палитра задаётся CSS-переменными, по две выверенные палитры (см. ADR-0011). Каждый блок имеет пару
`:root[data-theme="light"]` / `:root[data-theme="dark"]`:

- `templates/dashboard.html:23-50` — **основные токены** (`--bg`/`--panel`/`--ink`/`--muted`/`--line`/
  `--accent`/`--accent-soft`/`--ok`/`--warn`/`--warn-soft`/`--err`/`--err-soft`/`--chip`/`--reply-*`/
  `--sel`/`--shadow`/`--grid`) + фазовые тинты таймлайна `--ph-*`/`--ph-*-line`.
- `templates/dashboard.html:331-336` — **палитра подсветки diff** (`--syn-*`, `--gutter`).
- `templates/dashboard.html:~273` — третий `:root` цветов типов действий (`--t-tool/-bash/-mcp/-hook/
  -subtask`): **одинаков в обеих темах** (насыщенные, читаемы на обоих фонах) — без `data-theme`-вилки.
- `HUB_PAGE` в `scripts/server.py:1255-1270` — отдельная страница `/hub`, свои `:root[data-theme=…]`
  (палитра синхронизирована с дашбордом). Raw-строка, отдаётся дословно.

## Контракт темы

- `localStorage['theme']` хранит **предпочтение**: `'light' | 'dark' | 'system'` (дефолт `'system'`).
- `document.documentElement` несёт **разрешённый** `data-theme="light|dark"` — **никогда** `'system'`.
- `'system'` резолвится через `window.matchMedia('(prefers-color-scheme: dark)').matches` + подписка на
  `change` (тема следует за системной на лету, только пока предпочтение = `'system'`).
- **Один ключ для дашборда и хаба** → сквозной выбор темы хаб↔задача.

## Ключевые места

- `templates/dashboard.html:7-21` — **head-bootstrap** (`<script>` перед `<style>`): читает
  `localStorage`, резолвит, ставит `data-theme` до первой отрисовки. Резолв **продублирован** (head не
  видит функций из `<body>`); `try/catch` → fallback `light`.
- `templates/dashboard.html:535-560` — логика в основном `<script>`: `themePref()`/`resolveTheme()`/
  `applyTheme()` + `initTheme()` (клики `.seg`, listener `themeMql.change`). `applyTheme` ставит
  `data-theme` и подсвечивает активную кнопку (`.on`).
- `templates/dashboard.html:460-464` — **контрол** `.seg#theme-seg` в шапке (рядом с «Чат»):
  `data-theme-mode="light|dark|system"`, иконки ☀️🌙🖥 + `.seg-lbl` (скрываются на узкой ширине).
- `scripts/server.py:1237-1253` / `:1354-1370` — аналогичные bootstrap и логика в `HUB_PAGE`
  (продублированы — отдельный стрим, функции не шарятся).

## Инварианты

- **Обе палитры ПОЛНЫЕ и параллельные.** Тёмный блок **не наследует** светлый (после перехода с media
  на атрибут это самостоятельный селектор). Токен, добавленный в одну тему, **обязан** быть и в другой —
  иначе `undefined`. То же для diff-палитры и хаба.
- **`data-theme` — только `light|dark`** на `documentElement`. `'system'` существует лишь как
  предпочтение в `localStorage`, в DOM-атрибут не попадает.
- **Состояние темы — вне `#trace-summary`** (sig-гард ADR-0011): живёт на `documentElement` и контроле
  в шапке. Тик `/trace` (~3 с) затёр бы его внутри summary-DOM.
- **Резолв продублирован** (head-bootstrap ↔ `<body>`-функция; дашборд ↔ хаб) — правки синхронны.

## Подводные камни

- **Полнота тёмной палитры** — главная ловушка перехода с media на атрибут: ранее не-переопределявшиеся
  токены легко забыть (см. инвариант выше).
- **FOUC** — тему применяет только head-bootstrap; класть применение в `<body>`-JS поздно (вспышка).
- **`index.html` — копия шаблона.** Правки `templates/dashboard.html` попадают только в НОВЫЕ задачи;
  для текущей нужна повторная копия в `index.html` (сервер отдаёт его, `scripts/server.py:178`).
  Память проекта: живой companion-сервер крутит python из plugin-CACHE — проверять на репо-`server.py`.
- **Именная шероховатость:** soft-фон варнинга назван по-разному — дашборд `--warn-soft`, хаб
  `--awaiting-soft` (значения одинаковы). При правке тона помнить про два имени в двух файлах.

## Что НЕ темизируется (by design)

- Ролевая палитра таймлайна `ROLE_COLORS`/`roleColor` (ADR-0011) — бары с белым текстом, читаемы на
  обоих фонах.
- Песочные `iframe.mock` макетов (`background:#fff`) — самодостаточны (вне объёма брифа).
- `color:#fff` на цветных подложках (акцент/бары) — подложка тёмная в обеих темах.

## Как расширять

- **Новый цветовой токен:** добавить в **обе** вилки `:root[data-theme]` (и в хаб, если применимо).
- **Новый режим темы / другой контрол:** правится `.seg#theme-seg` + `applyTheme`/`initTheme`; помнить
  про дублирование head-bootstrap и хаба.
- **Сменить дефолт первого захода:** `localStorage.getItem('theme') || '<default>'` в head-bootstrap
  и в `themePref()` (обе копии, в дашборде и хабе).
