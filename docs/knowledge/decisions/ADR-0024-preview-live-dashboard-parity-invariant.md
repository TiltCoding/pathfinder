# ADR-0024 — Паритет превью ↔ реальный дашборд как проверяемый инвариант

- **Статус:** принято (2026-06-27)
- **Контекст:** `/improve overall-3`, feat-2 (cand-19) — обязательный seed-кандидат человека:
  «вид превью-шаблона ВСЕГДА точно соответствует реальному дашборду при запуске агентов».

## Проблема

Превью (`scripts/preview.py`) и живой прогон рендерят дашборд из формально **разных источников**:

- превью копирует `templates/dashboard.html` и поднимает `server.py` строго из **репо**
  (`scripts/preview.py:46-47`, штамп `:135`, спавн `:182`);
- живые `/feature`//ask//improve//design копируют шаблон из **плагин-кэша**
  (`${CLAUDE_PLUGIN_ROOT}/templates/dashboard.html`, `skills/*/feedback-loop.md`), и companion-сервер
  тоже может крутиться из кэша.

Сам штамп `index.html` — это `shutil.copyfile` шаблона (идентичность по построению), но **ничем не
проверялось**, что так и останется: замена копира на трансформацию (минификация, инъекция токена)
расходила бы превью с живым дашбордом молча. Плюс ось **репо vs плагин-кэш**: правки репо доходят до
live только после refresh (ловушка «плагин-кэш», `dashboard-i18n.md`).

## Решение

Сделать паритет **проверяемым**, не «на честном слове», тремя дешёвыми мерами (без новой рантайм-логики
дашборда):

1. **Тест паритета** (`tests/test_preview_fixtures.py::PreviewParityTest`): прогоняет
   `preview.install()` в изолированном tempfile-корне (monkeypatch `TASKS_DIR`, нейтрализация
   `_sweep_legacy`, чтобы не трогать живой store) и утверждает `sha256(index.html) == sha256(шаблона)`
   для каждой фикстуры. Любая будущая трансформация копира падает в CI. Превращает «идентично по
   построению» в **закреплённый инвариант**.
2. **Self-report источника**: `/health` отдаёт поле `source` = `os.path.realpath(__file__)` работающего
   `server.py` (`scripts/server.py`). `preview._check_source` дочитывает `/health` и **предупреждает**,
   если поднятый сервер не из репо (`!= ROOT/scripts/server.py`) — делает молчаливый дрейф репо↔кэш
   видимым. Поле аддитивно, `/health` обратносовместим.
3. **Документация** (`dashboard-i18n.md`): зафиксирован шаг «refresh плагина из репо ПЕРЕД сверкой с
   live» (или запускать live тоже из репо-`server.py`), чтобы `CLAUDE_PLUGIN_ROOT` совпадал с репо.

## Почему так

- **Тест, а не рантайм-гард**: паритет — свойство сборки/копира, его место в CI, а не в горячем пути.
- **sha256, а не «примерно»**: байтовое равенство ловит любую трансформацию без ложных срабатываний.
- **`source` в `/health`, а не новый эндпоинт**: переиспользуем существующий self-report (pid/port/root,
  ADR-0017), нулевая новая поверхность.
- **Изоляция теста от live-store**: monkeypatch `TASKS_DIR`/`_sweep_legacy` — тест оффлайн и
  кросс-платформенный, не засоряет `.workflow/`.

## Последствия

- Замена `copyfile` на трансформацию ловится CI, а не глазами в проде.
- Расхождение репо↔кэш видно как предупреждение превью, а не как «загадочный старый дашборд».
- Известный остаток: per-task `index.html` — снимок при старте задачи (вторая ось дрейфа); устранение
  «рендер из шаблона на лету» вынесено в отдельный кандидат (`stale per-task index.html`), не входит сюда.

Связь: ADR-0017 (`/health` self-report), `dashboard-i18n.md` (ловушка плагин-кэша).
