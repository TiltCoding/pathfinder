# ADR-0004: Подсветка синтаксиса diff встроенным токенайзером без CDN

- **Дата:** 2026-06-10
- **Статус:** принято
- **Задача:** changed-files-tree-view

## Контекст

Вкладка «Изменения» показывала diff без подсветки синтаксиса. Требование — «код как код». Очевидный
путь — подключить highlight.js/Prism с CDN, но дашборд работает в окружении без гарантии внешней сети, и
проект уже сознательно рендерит markdown (`md()`) и весь UI без внешних зависимостей.

## Решение

Собственный лёгкий построчный токенайзер `highlightCode(line, lang)` (`templates/dashboard.html:540`) +
определение языка по расширению `langFromPath(path)` (`:520`). Минималистичный набор токенов (решение
q2=A): строки, комментарии, числа, ключевые слова языка, имя функции. Языки — реальные в репозитории:
`py/js/json/html/md/yaml/yml/css/sh/bash`; неизвестное расширение → без подсветки, только diff-семантика.
В `renderDiff(text, lang)` (`:1020`) тело строки прогоняется через токенайзер **после снятия** ведущего
`+`/`-`/` ` префикса, фон add/del/hunk/meta сохранён.

## Почему

- **Нет внешней сети — нет CDN.** Встроенный движок работает офлайн и не добавляет зависимостей; та же
  политика, что у `md()`.
- **Построчность под формат diff.** Diff приходит строками — построчный токенайзер естественно ложится на
  рендер и не требует парсить файл целиком.
- **Безопасность по умолчанию.** Токенайзер работает **поверх `esc()`** и не вставляет неэкранированный
  пользовательский текст — нет XSS (проверено в ревью).
- **Префикс вне токенайзера.** Снятие `+`/`-` до подсветки — иначе они путаются с операторами.

## Последствия

- Подсветка минимальная: не различает все конструкции языка (достаточно для читаемости diff). Богатая
  подсветка отклонена осознанно (q2).
- Новый язык = правка `langFromPath` + ветка ключевых слов в `highlightCode`, без внешних либ.
- Движок живёт на фронте; backend `/changes?file=` остаётся отдающим сырой текст diff.
