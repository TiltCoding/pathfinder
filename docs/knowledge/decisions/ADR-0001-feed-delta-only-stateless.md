# ADR-0001: Лента трейсинга — delta-only / stateless, склейка start↔end на клиенте

- **Дата:** 2026-06-10
- **Статус:** принято
- **Задача:** realtime-agent-tracing

## Контекст

Вкладке «Трейсинг» нужен почти реал-тайм поток действий агента. Объём `telemetry.jsonl` при ловле всех
инструментов растёт на порядок. Существующий `GET /trace` читает файл целиком и джойнит транскрипты —
тяжёлый, с кэшем 3 с. Развилки: (а) расширять `/trace` курсором `sinceLine=N` или сделать отдельный
эндпоинт; (б) склеивать `tool.start`/`tool.end` в спаны на сервере (нужно держать состояние между тиками)
или отдавать плоские дельты и склеивать на клиенте; (в) курсор по номеру строки или по байтовому оффсету.

## Решение

Отдельный лёгкий эндпоинт `GET /trace/feed?slug&since=<byteOffset>`. Сервер на каждом тике читает **только
хвост** файла за `since` байт (`_iter_lines_from`, `scripts/_aipf.py:381`), возвращает плоские дельта-записи
`tool.*` и `nextOffset`. Сервер **не хранит состояния** между запросами и **не склеивает** start/end —
клиент инкрементально строит модель по `spanId` и сам выводит `running`. Курсор — **байтовый оффсет**
(`f.tell()`), не номер строки. `build_feed` — `scripts/_aipf.py:432`, `_trace_feed` — `scripts/server.py:332`.

## Почему

- **O(новых строк) вместо O(всего файла).** При росте телеметрии полное чтение на каждый тик неприемлемо;
  оффсетный seek/tell даёт инкрементальность напрямую.
- **Байтовый оффсет точно ложится на `f.seek()/f.tell()`** и переживает докатку строк; номер строки
  требовал бы хранить счётчик и был бы хрупок при конкурентной дозаписи.
- **Stateless-сервер** проще и не конфликтует с фоновым форвардером и кэшами; вся «память» ленты живёт в
  браузере, где она и нужна (скролл, раскрытые блоки).
- **Склейка на клиенте** естественна: UI всё равно держит инкрементальную модель по `spanId`, умеет
  показывать `running` для висящих `tool.start` без `end` (прерванный инструмент — валидный случай).
- Отдельный эндпоинт **не смешивает** живую лёгкую ленту с тяжёлой посмертной моделью `/trace` и не ломает
  её контракт (сводка/гант остаются как есть).

## Последствия

- Клиент обязан хранить `lastOffset` и накопленную модель спанов; рендер — диффом по `spanId`, иначе
  сбрасываются скролл и раскрытые блоки.
- Кэш `/trace/feed` короткий (≤1 с), чтобы лента была свежей.
- При `since>0` атрибуция lane для tool-события, чей `subagent.start` остался до курсора, деградирует до
  `orchestrator` — клиент корректирует по своей модели (см. ADR-0003).
- Поддерживать инвариант: формат старых событий менять нельзя (курсор Langfuse, см. область телеметрии).
