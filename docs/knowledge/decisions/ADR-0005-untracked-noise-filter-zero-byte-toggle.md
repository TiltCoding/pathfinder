# ADR-0005: Фильтр untracked-мусора по 0 байт + тумблер «только tracked / все»

- **Дата:** 2026-06-10
- **Статус:** принято
- **Задача:** changed-files-tree-view

## Контекст

В список «Изменений» лезли stray-файлы (`-`, `Заполняется`, `Одна`, `Сначало`, `Это`) — реальные пустые
untracked-файлы в корне репозитория, засоряющие дерево. Нужно показать только осмысленные изменения, не
теряя при этом полноту (иногда важно увидеть все untracked). Развилка: жёстко прятать весь untracked,
жёстко показывать всё, или дать управление.

## Решение

Эвристика «мусора» — предикат `_is_noise(relpath, status)` (`scripts/server.py:546`): скрывается
**untracked-файл нулевого размера** (q1=A). Tracked-изменения и непустые untracked не трогаются;
`.gitignore` уже уважается git, не дублируем. В записи файла есть поле **`untracked: true`**, чтобы фронт
без второго запроса применял **segmented-тумблер** «Только tracked / Все» (q3=A) — по умолчанию «Только
tracked», untracked-мусор скрыт. Untracked-каталоги при этом честно развёрнуты благодаря `-uall`
(`status --untracked-files=all`).

## Почему

- **0 байт — надёжный признак мусора** в этом репозитории: содержательные новые файлы непусты, а stray —
  именно пустышки. Простая, объяснимая эвристика без списков исключений.
- **Тумблер вместо жёсткой политики.** Пользователь сам решает «чисто vs полно», не навязываем ни одно из
  крайних поведений.
- **Поле `untracked` на фронте.** Фильтрация тумблера — без повторного запроса; backend остаётся с одним
  контрактом `/changes`.

## Последствия

- Непустой, но бессмысленный untracked-файл фильтр не скроет (по дизайну — только 0 байт). Если появится
  другой класс мусора — расширять `_is_noise`, а не фронт.
- Состояние тумблера (`changesShowAll`) держится в модуле фронта и переживает polling-перерисовку.
- Эвристика применяется только к `status == "added"` (untracked); tracked-изменения всегда видимы.
