# ADR-0002: Matcher хуков `.*` + фильтр шума `TRACE_TOOLS` в Python

- **Дата:** 2026-06-10
- **Статус:** принято
- **Задача:** realtime-agent-tracing

## Контекст

Раньше хуки матчили узкий набор: `PreToolUse` = `Task`, `PostToolUse` = `Task|Write|Edit|MultiEdit|
NotebookEdit`. Для живой ленты нужны тайминги начала/конца **значимых** инструментов (Bash/Read/Grep/Glob/
Edit/Write), но не весь шум (TodoWrite и пр.). Развилка: ограничивать набор **в matcher** (regex по
именам инструментов в `hooks/hooks.json`) или ловить всё через `.*`, а фильтровать **в Python**.

## Решение

`PreToolUse` и `PostToolUse` matcher = `.*` (ловят все инструменты, `hooks/hooks.json:38,50`). Фильтр
значимого набора вынесен в `telemetry_hook.py`: множество `TRACE_TOOLS` (`scripts/telemetry_hook.py:37`).
Инструмент не из набора падает сквозь все ветки `build_event` и хук выходит **без записи**.

## Почему

- **Одно место правды для набора инструментов** — Python-множество, а не regex, размазанный по двум секциям
  JSON. Менять набор = править одну строку, без риска рассинхрона Pre/Post матчеров.
- **Гибкость фильтрации.** В Python легко исключить под-агентов (`Task`/`Agent` идут как богаче `subagent.*`)
  и применять разную логику к `file.touch`/`tool.*` — regex так не умеет.
- **Цена принята осознанно.** `.*` запускает `python3` на каждый инструмент, включая нетрейсимые; они сразу
  выходят без записи. Лишние запуски — компромисс ради простоты и единого фильтра. Правило «любая ошибка →
  exit 0» (ADR неявно, см. conventions) защищает горячий путь.

## Последствия

- Накладные расходы на запуск интерпретатора для нетрейсимых инструментов (TodoWrite и т.п.). Если это
  станет проблемой производительности — сузить matcher или ввести «быстрый выход» в начале `main`.
- Изменение набора трейсимых инструментов — только через `TRACE_TOOLS`; matcher трогать не нужно.
- Любой новый инструмент CC автоматически проходит через хук; чтобы он попал в ленту, его надо добавить в
  `TRACE_TOOLS` (+ опц. `_TRACE_ARG_FIELD`).
