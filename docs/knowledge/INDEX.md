- [decisions/ADR-0008-feedback-on-existing-contract-zero-server.md](decisions/ADR-0008-feedback-on-existing-contract-zero-server.md) — свой ответ как `answer` вне `options` (перебивает опцию, один answer/вопрос) и коммент к варианту как `comment` с `blockId=vr.id` (footer всегда); 0 правок сервера — бэкенд агностичен к содержимому items
- [decisions/ADR-0009-transcript-attribution-and-actions-endpoint.md](decisions/ADR-0009-transcript-attribution-and-actions-endpoint.md) — поправка к ADR-0003: достоверная по-агентная атрибуция через транскрипт + `meta.json.toolUseId` (мост спан→транскрипт), ленивый `/trace/actions` вместо утяжеления `build_trace`, MCP в ленте дозаписью `kind/server/mcpTool`; lane остаётся best-effort, у задачи бывает несколько session_id
- [decisions/ADR-0010-shared-store-symlink-worktree.md](decisions/ADR-0010-shared-store-symlink-worktree.md) — общий store через симлинк `<worktree>/.workflow → <main>/.workflow` (vs env `AIPF_WORKFLOW_ROOT` vs реестр `runs.json`), `worktreePath`/`branch` в state для per-worktree diff, per-session `active.json` обязателен, токены вне кросс-задачного агрегата, хаб — отдельная страница `/hub` (не вкладка)
- [decisions/ADR-0011-timeline-variant-c-token-rate-phases.md](decisions/ADR-0011-timeline-variant-c-token-rate-phases.md) — таймлайн «Трейсинг» Вариант C (клиентский `renderGantt`, 0 правок сервера): обзор по **производству токенов** (`rate=out/durationMs`, агрегация по бакетам — аппроксимация), разбивка на **фазы workflow** (фон-полосы + «вне фазы»; честно: `phase` разрежён, `workstream` хук не пишет), фикс палитры через `roleKey` (namespace-префикс `ai-pathfinder:wf-*`), подводный камень `sig`-гарда перерисовки
- [decisions/ADR-0012-improve-consensus-panel-deterministic-aggregation.md](decisions/ADR-0012-improve-consensus-panel-deterministic-aggregation.md) — консенсус `/improve`: рой 7 scout по призмам → консолидация+дедуп оркестратором (`cand-1…N`) → панель 3 vote (независимая оценка всего списка) → **детерминированная агрегация оркестратором** (`score=(mean(imp)−w_e·mean(eff)−w_r·mean(rsk))·mean(conf)/3`, `w=0.5`, «согласие»=доля keep, топ-K 6–8); почему панель+арифметика, а не «консенсус в одном агенте» (субагенты не спавнят субагентов; честность голосов); расширяет паттерн судейской панели ADR-0006/0007
- [decisions/ADR-0013-improve-feature-pick-reuse-zero-server.md](decisions/ADR-0013-improve-feature-pick-reuse-zero-server.md) — выбор фич `/improve` поверх существующего контракта: `feat-K` (карточка `planBlocks` ↔ `questions[choice]` с тем же id) + сигнал `approve-plan`, **0 правок server.py/HTML**; дефолт «нет ответа=Пропускаем», порядок Submit→Approve (`draft.json` не READABLE); развилка A (выбран) vs B (аддитивный чеклист, апгрейд); связь с ADR-0008. **Уточнение (2026-06-15, `improve-overall`):** оркестратор с файловым доступом МОЖЕТ прочитать выбор напрямую из `draft.json`, если Submit пропущен, а draft полон/однозначен (fallback к Submit→Approve, не отмена контракта)
- [decisions/ADR-0014-improve-sequential-feature-queue.md](decisions/ADR-0014-improve-sequential-feature-queue.md) — DISPATCH `/improve` = **последовательная очередь** `.workflow/dispatch-queue.json` + дренаж через `/feature` по одной фиче в свежем контексте (вместо параллельного worktree-фан-аута); `/feature` получил **queue-mode** (пропуск INTAKE, бриф из очереди, на DONE → `/clear`+`/feature` или `/loop /feature`); `/improve` не запускает `/feature` сам (чистота контекста); worktree-фан-аут остаётся opt-in; контракт `skills/improve/dispatch-queue.md`
- [decisions/ADR-0015-theme-toggle-data-attribute-localstorage.md](decisions/ADR-0015-theme-toggle-data-attribute-localstorage.md) — переключение темы дашборда/хаба (2 режима) через `data-theme`-атрибут `<html>` + явный выбор `localStorage['theme']` (`light|dark`; отсутствие/legacy `system` = следуем системе, резолв через matchMedia + `change` пока выбор не сделан); контрол — **иконка-кнопка** `#theme-btn` (☀️/🌙), не `.seg`; триггер сменён с `@media` на атрибут (5 блоков), FOUC-bootstrap в `<head>` (резолв продублирован); токены `--err`/`--err-soft`; общий ключ хаб↔задача; **тёмный блок не наследует светлый → полнота палитры обязательна**; именная шероховатость `--warn-soft`/`--awaiting-soft`. Поправка v0.14.x: с 3-режимного segmented на 2-режимную иконку под фактический выбор. Связь с ADR-0011 (две темы) и ADR-0005 (`.seg` для темы НЕ используется)
- [decisions/ADR-0016-ask-readonly-qa-over-existing-contract.md](decisions/ADR-0016-ask-readonly-qa-over-existing-contract.md) — `/ask` (read-only Q&A) поверх существующего контракта: текст=`summary`/`planBlocks`, визуализации=`demo`/`mockups` (sandbox-iframe, SVG из коробки), чат=`chat.jsonl`, хаб — автоматически; единственная правка кода — append-only бейдж `state.json.kind:"ask"` в `HUB_PAGE` (~4 строки `server.py`, HTML не тронут); мини-рой read-only `ask-researcher` по граням + консолидация оркестратором (расширяет рой `/improve`, но **без** голосования/очереди); синтез/рисование — у оркестратора (инвариант «субагенты не спавнят субагентов»); нетерминальная `ANSWER` + авто-DONE ~24 ч; `ask-researcher` без `model:` (как `wf-*`). Связь с ADR-0006 (синтез/модель), ADR-0008/0013 (агностичный бэкенд), ADR-0010 (хаб/общий store), ADR-0012/0014 (рой/очередь `/improve` — чем отличается)

- [decisions/ADR-0017-server-singleton-heartbeat-stable-port.md](decisions/ADR-0017-server-singleton-heartbeat-stable-port.md) — надёжность companion-сервера: **идемпотентный singleton** (`server_is_live` → `reusing live server`, не плодим дубликаты), `ts`-heartbeat + `SERVER_STALE_SECS` (труп с переиспользованным pid), стабильный порт `port_for_root` (sha1 корня), `root` в `server.json`/`/health`, reap осиротевших (`discover_servers`/`gc_targets`/`reap_servers`, `--gc`), очистка `server.json` на выходе (`install_shutdown_cleanup`/`clear_server_info`); почему не демонизация (серверы и так переживали сессию — копились). Связь с ADR-0010 (store/хаб)
